메타데이터

데이터를 설명하는 데이터
head태그 안에 들어가있다. 검색엔진은 보통 head태그 안에 있는 메타데이터를 사용해서 그 페이지 안의 내용을 정리하여 노출시킨다.

title 태그

웹페이지를 설명하는 태그.

  • 태그 안쪽에 태그가 있더라도 완전히 무시를 함.
  • 하나의 타이틀 안에는 단 하나의 타이틀 태그만 있다.
  • 단순 명사만 나열한 title은 피해야함. 검색알고리즘이 기피함.

meta태그

  • 빈요소
  • 이름과 값이 한쌍이다.
  • 정보가 여러개일 경우 메타태그를 여러개 적어서 설명.
  • 필수값은 아니고 추가정보.

meta태그 속성

  • application-name : 웹페이지에서 구동중인 애플리케이션의 이름
  • description : 페이지에 대한 짧고 정확한 요약.
    여러 브라우저는 즐겨찾기의 기본설명값으로 description을 사용한다.
  • keywords : 인스타그램의 해시태그정도로 기억하자.
  • Referrer : 링크로 이동할 때 내가 어디에서 왔는지 알려주는 태그

문자 인코딩

charset

head 첫번째 요소로 넣어주는 것이 좋다. 페이지를 어떻게 인코딩 할 것인지 정한다.(보통 전세계 언어를 지원하는 UTF-8을 넣는다고 함.)

뷰 포트

name=viewport

초기 사이즈에 대한 힌트.
content에 내용을 적는다.

  • width=device-width : 웹사이트 넓이를 디바이스 넓이에 맞춰라.
    보통 width를 설정하면 height는 따라오기 때문에 height는 따로 정하지 않는 편.
  • initial-scale : 장치너비와 뷰포트 너비의 비율을 정의한다. 보통 1.0으로 설정
  • user-scalable : 웹페이지를 확대할 수 있게끔 할거냐 말거냐. 기본값은 yes.
  • maximum-scale, minimum-scale : 확대나 축소의 제한값

mime 타입

불러온 외부소스의 타입을 알려주는 속성.

style태그

이런식으로 별도의 파일 없이 css스타일을 적용시킬 수 있다.

말고 head쪽에 style태그 안에 css를 적용시킬 수도 있다.

같은 위치에 css명령을 내리면 나중에 명령한 css가 적용된다.

script태그

html에서 JavaScript를 사용하게 해준다.
head태그에 script태그가 있다면 script를 해석하는 동안에는 밑에있는 코드들은 렌더링되지않는다.
그래서 엄청 오래걸림. script태그는 왠만하면 body의 마지막 부분에 넣어야 html이 먼저 렌더링 되고 시간이 오래 걸리는 js는 마지막에 해석된다.

p태그

컨테이너 역할을 하는 본문 태그.
이미지나 입력 폼 등 관련있는 콘텐츠는 뭐든 넣을 수 있다.

p태그 사이에는 폰트사이즈 만큼의 공백이 생긴다.
빈 공간을 위해 p태그를 사용하면 스크린리더가 문단의 존재는 인지하지만 읽을 내용이없어서 혼란을 느낄 수 있다.
